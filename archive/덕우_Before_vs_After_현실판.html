핵심 제약 조건 — 이걸 모르면 설계가 다 틀어짐

  • 유플랫 API 없음 — 데이터 추출은 엑셀 수동 다운로드뿐. 가공 결과를 다시 넣는 것도 수동.
  • 샵마인 API 없음 — CS/주문/정산 데이터도 엑셀 수동 추출뿐.
  • 유플랫 우회하면 깨지는 것들 — 재고 실시간 연동, 매일 가격 변동 반영, 주문처리(UOS), 발주, 배송. 이걸 다 직접 구현해야 함.
  • 28.5만건 전량 순환은 비현실적 — 실제 월 순환량 기준으로 봐야 함.

✕ 유플랫 월 79.8만원 계속 지출
✕ "풀 자동화"는 원천적으로 불가
B. 유플랫 제거 + 네이버 API 직접 (야심적 선택)
✅ 가공~등록까지 완전 자동화 가능
✅ 유플랫 월 79.8만원 절약
✕ 51곳 도매처 크롤링을 직접 구축해야 함 (대공사)

C. 유플랫 엑셀 대량 임포트 가능 여부 확인 (절충안)
✅ 유플랫이 엑셀 "가져오기"를 지원하면 → 가공 자동 + 업로드 자동
✅ 기존 크롤링/재고/주문 유지하면서 가공만 외부 처리
✕ 유플랫이 지원하는지 아직 확인 안 됨
