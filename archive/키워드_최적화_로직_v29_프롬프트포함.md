## 전체 흐름 (Excel 모드)

```
상품명 + 이미지URL 입력
  → Step A1: Gemini AI 키워드 생성 (55~80개)
  → Step A2: Gemini 수식어 추출 + 기계적 조합 (NEW)
  → Step B: 네이버 1차 확장 (A1+A2 → 연관키워드 전체 수집)
  → Step C: 네이버 2차 확장 (B에서 새로 나온 키워드 → 재확장, 최대 200개)
  → Step D: 금칙어/브랜드 필터
  → Step E: 검색량 필터 (PC≥10 OR mobile≥10, total≤50,000)
  → Step F: 이미지 필터 (Gemini 비전, 85점 이상)
  → 최종 키워드 리스트 + KQS 등급
```

## Step A1: Gemini AI 키워드 생성 (기존)

### A1-1. 상품명 파싱 (extractCoreKeywords)

입력 예시: `[덕우무역] EVA 논슬립 선반매트 500mm 화이트`

5단계 순차 정규식 처리:

1. **대괄호 제거** — `[덕우무역]` 등 브랜드/판매자 표기 제거
2. **모델번호 제거** — 영문+숫자 조합 (KSG-77D, AB-1234, X100T 등)
3. **영문 단어 필터** — 단독 영단어 제거. 단, 허용 목록은 유지:
   - 허용: pro, max, mini, plus, lite, ultra, set, kit, led, usb, dc, ac
   - EVA, PVC 등 소재명도 현재는 제거됨 (허용 목록에 없으므로)
4. **용량/수량 제거** — 500mm, 800ml, 1.5L, 100매, 10개, 3P 등
5. **토큰화** — 2자 이상 한글 단어만 추출, 중복 제거

파싱 결과: `["논슬립", "선반매트", "화이트"]`

### A1-2. Gemini 프롬프트 (원문 그대로)

```
당신은 네이버 스마트스토어 SEO 전문가이자 뉴스·기사 키워드 분석가입니다.

상품명: "{상품명 원문}"

핵심 키워드 힌트: [{파싱된 토큰들}]
위 핵심 키워드를 기반으로 관련 검색어를 생성하세요.
상품명이 길거나 모델번호가 포함되어 있어도, 소비자가 실제 검색할 일반 키워드를 만들어야 합니다.

이 상품을 네이버에서 검색할 때 소비자가 실제로 입력할 수 있는 다양한 키워드를 생성하세요.
일반 소비자 검색어뿐 아니라, 뉴스기사·보도자료·전문리뷰에서 사용하는 격식체 표현도 반드시 포함하세요.
또한 소비자가 자신의 문제나 고민을 검색하는 패턴, 특정 대상이나 상황에 맞는 검색어도 포함하세요.

규칙:
1. 원본 상품명에 이미 있는 단어 조합은 제외
2. 카테고리당 5~8개 (총 55~80개 내외)
3. 띄어쓰기 없이 붙여쓰기
4. 동의어 과도 금지
5. 브랜드명/모델번호는 제외하고, 일반 소비자가 검색할 범용 키워드만 생성
6. 뉴스·기사체 키워드는 네이버뉴스, 블로그리뷰, 보도자료에서 실제 사용되는 표현 수준으로 생성

카테고리:
1.정식명칭변형 2.기능효과 3.보호목적 4.구어체검색어 5.일반명칭 6.소재형태 7.용도시즌
8.뉴스·기사체동의어 (보도자료·리뷰기사에서 쓰는 격식체 표현, 예: 인체공학적디자인, 장거리운전피로경감)
9.전문용어·스펙표현 (제품스펙리뷰·비교기사에서 쓰는 기술적 표현, 예: 고밀도메모리폼, 통기성소재)
10.문제·고민형 (소비자가 자기 문제를 검색하는 패턴, 예: 장거리운전팔저림, 운전중팔꿈치통증, 허리아파서)
11.대상·상황형 (누구한테/어떤 상황에서 쓰는지, 예: 택시기사선물, 아버지날선물, 장거리출퇴근필수)

JSON 배열로만 반환. 예: ["차량커버", "방수차커버"]
```

### A1-3. Gemini 호출 설정

- 모델: gemini-2.5-flash (실패 시 gemini-2.5-flash-lite fallback)
- 각 모델 최대 3회 retry (429→3/6/9초 대기, 500+→2/4/6초 대기)
- temperature: 1.0 (다양성 극대화)
- maxOutputTokens: 4096
- 프롬프트 1개, 호출 1회 (카테고리별 분리 호출 아님)

### A1-4. 응답 파싱

1. **thinking 파트 제거** — Gemini 2.5는 응답에 `{thought: true}` 파트가 섞여 나옴. 뒤에서부터 스캔해서 thought가 아닌 마지막 텍스트만 추출
2. **JSON 배열 추출** — 코드블록 마커 제거 후 `[...]` 패턴 매칭 → JSON.parse

### A1-5. 후처리 및 병합

1. Gemini 결과에서 문자열 2자 이상만 필터
2. A1-1에서 뽑은 핵심 토큰(논슬립, 선반매트, 화이트)을 뒤에 추가 (Gemini가 놓칠 수 있으므로)
3. Set으로 중복 제거
4. 최대 100개 cap

### A1 출력
- 최대 100개 키워드 배열

## Step A2: 수식어 추출 + 기계적 조합 (NEW — v29)

> **목적:** Gemini가 완성된 키워드를 만들면 빠지는 조합이 생김. 대신 **수식어(modifier)만 뽑고, 코드가 전수 조합**하면 빠짐없이 만들 수 있음.

### A2-1. 핵심 상품어 결정

A1-1에서 추출한 토큰 중 **상품 본질을 나타내는 1~2개**를 핵심 상품어로 선정.

예시:
- `[덕우무역] EVA 논슬립 선반매트 500mm 화이트` → 핵심 상품어: `["선반매트"]`
- `밀워키 M12 PN23GS-0X0 충전 실타카 12V` → 핵심 상품어: `["실타카"]`
- `접이식 경량 알루미늄 캠핑 테이블` → 핵심 상품어: `["캠핑테이블", "테이블"]`

결정 방법: Gemini에게 "이 상품명에서 핵심 상품 카테고리 명사를 1~2개만 뽑아줘"라고 별도 요청하거나, extractCoreKeywords 결과에서 가장 구체적인 명사를 자동 선택.

### A2-2. Gemini 수식어 프롬프트 (NEW)

```
당신은 네이버 쇼핑 키워드 전문가입니다.

상품명: "{상품명 원문}"
핵심 상품어: [{핵심 상품어}]

이 상품에 붙일 수 있는 **수식어를 단어 하나씩만** 나열하세요.
수식어란 핵심 상품어 앞뒤에 붙어서 검색 키워드를 만드는 단어입니다.

카테고리별로 생성:
1. 용도 (목공, 인테리어, DIY, 가정용, 업소용, 현장...)
2. 방식/동력 (충전, 무선, 전동, 에어, 수동...)
3. 소재/규격 (스테인리스, 알루미늄, 대형, 소형, 미니...)
4. 대상/상황 (초보, 입문, 전문가, 선물, 이사...)
5. 기능/특징 (경량, 접이식, 방수, 논슬립, 휴대용...)
6. 동의어/유사어 (타카→타카기, 매트→패드→시트...)

규칙:
- 반드시 **한 단어씩만** (띄어쓰기 없이)
- 브랜드명/모델번호 제외
- 총 40~60개
- JSON 배열로만 반환

예시: ["충전", "무선", "목공", "DIY", "소형", "경량", "전동"]
```

### A2-3. 호출 설정

- 모델: gemini-2.5-flash (A1과 동일 fallback)
- temperature: 0.8 (A1보다 약간 낮게 — 수식어는 정확도 필요)
- maxOutputTokens: 2048
- 호출 1회

### A2-4. 기계적 조합 (generateCompounds)

Gemini가 반환한 수식어 배열과 핵심 상품어를 **코드로** 전수 조합:

```
입력:
  핵심 상품어: ["실타카"]
  수식어: ["충전", "무선", "목공", "DIY", "소형", "경량", "전동", "에어리스", "브래드", "핀"]

조합 규칙:
  1. 2단어 조합: 수식어 + 핵심어 (모든 수식어 × 핵심어)
     → "충전실타카", "무선실타카", "목공실타카", ...
  
  2. 3단어 조합: 수식어A + 수식어B + 핵심어 (상위 수식어 간 조합)
     → "충전무선실타카", "목공전동실타카", "DIY충전실타카", ...
  
  3. 역순 조합: 핵심어 + 수식어
     → "실타카충전", "실타카무선", ...

출력: 중복 제거 후 전체 조합 배열
```

### A2-5. 조합 제한

- 2단어 조합: 수식어 × 핵심어 = 최대 60 × 2 = 120개
- 3단어 조합: 상위 20개 수식어 간 조합 = C(20,2) × 핵심어 = 최대 380개
- 역순: 핵심어 × 수식어 = 최대 120개
- **총 cap: 500개** (이 중 네이버 검색량 없는 것은 Step E에서 제거)

### A2-6. A1 + A2 병합

- A1 결과 (최대 100개) + A2 결과 (최대 500개) = 합집합
- Set으로 중복 제거
- 이 병합 결과가 Step B의 입력이 됨

## Step B: 네이버 1차 확장

### 동작

A1+A2 결과를 네이버 광고 API `/keywordstool`에 5개씩 배치로 넣어 **연관 키워드 전체**를 수집한다.

예: "충전실타카"를 넣으면 네이버가 "실타카추천", "미니실타카", "타카핀" 등 수십 개 연관 키워드를 반환.

### 캐시 시스템 (v28~)

- `keyword_cache` Supabase 테이블에서 먼저 조회
- 캐시에 있는 키워드는 네이버 API 호출 **생략**
- 캐시에 없는 키워드만 네이버 API 호출 → 결과를 캐시에 저장
- **효과:** 같은 키워드 재검색 시 API 사용량 대폭 감소

### API 호출

```
GET https://api.searchad.naver.com/keywordstool
  ?hintKeywords={5개 콤마 구분}&showDetail=1

Headers:
  X-Timestamp: {밀리초}
  X-API-KEY: {네이버 광고 API 키}
  X-Customer: {고객 ID}
  X-Signature: HMAC-SHA256(timestamp.GET./keywordstool)
```

### 출력

- B_all: 네이버가 반환한 모든 연관 키워드
- B_new: A에 없던 새로운 키워드만 필터
- 배치 간 300ms 딜레이

## Step C: 네이버 2차 확장

### 동작

Step B에서 **새로 발견된 키워드(B_new)**를 다시 네이버에 넣어 한 번 더 확장.

### 제한

- B_new 중 최대 **200개**만 2차 확장 입력으로 사용
- 캐시 시스템 동일 적용
- C_new: A+B에 없던 새로운 키워드

### 왜 2라운드?

1라운드만 하면 "충전실타카" → "타카핀" 까지는 나오지만, "타카핀" → "브래드핀규격" 같은 2단계 연관어는 못 잡음.
2라운드를 돌리면 키워드 풀이 2~3배 확장됨.

### 병합

- allNaverData = naverData1 ∪ naverData2 (Map merge)
- allPool = A ∪ B_all ∪ C (전체 키워드 풀, 중복 제거)

## Step D: 금칙어/브랜드 필터

### 필터 1: 광고성 금칙어 (하드코딩)

```
추천, 세일, 할인, 강추, 인기, 베스트, 핫딜, 최저가, 가성비, 순위, 비교,
최고급, 최신형, 대박, 핫한, 트렌디, 구매, 판매, 이벤트, 쿠폰, 적립,
사은품, 특가, 한정판매, 기간한정, 무료배송, 당일배송, 빠른배송, 퀵배송,
공식스토어, 정품, 직영, 본사직송, 공식판매, 좋은, 나쁜, 최고의, 최상급
```

키워드 안에 위 단어가 **포함**되면 제거 (부분 일치).

### 필터 2: 판매자 브랜드

상품명의 **첫 번째 대괄호** 내용을 판매자 브랜드로 인식.
- `[덕우무역] EVA 선반매트` → "덕우무역"이 포함된 키워드 제거

### 필터 3: IP 브랜드/모델명 (500개+)

자동차, 가전, 디지털, 패션, 스포츠, 화장품, 식품, 가구, 주방, 건강, 반려동물, 시계, 악기, 완구, 문구, 공구, 렌탈 등 **17개 카테고리**에 걸친 IP 브랜드 목록.

**예외 규칙:** 상품명 자체에 해당 브랜드가 포함된 경우는 허용.
- 상품명이 "밀워키 실타카"이면 → "밀워키"가 포함된 키워드는 제거 안 함
- 상품명이 "논슬립 매트"이면 → "다이슨매트" 같은 키워드는 제거

## Step E: 검색량 필터

### 조건

allNaverData에서 검색량 데이터를 조회하여:

```
통과 조건:
  (PC 월간 검색량 ≥ 10  OR  모바일 월간 검색량 ≥ 10)
  AND
  (PC + 모바일 합계 ≤ 50,000)
```

- "< 10" 문자열 → 0으로 처리
- 검색량 0인 키워드 → 제거 (네이버에서 실제 검색되지 않는 조합)
- 50,000 초과 → 제거 (너무 광범위한 키워드)

### KQS (Keyword Quality Score) 산출

```
KQS = (검색량 × CTR × 2) / (경쟁도 × 광고깊이)

경쟁도 매핑: 높음=3, 중간=2, 낮음=1
광고깊이: 최소 0.5 (0일 경우)
CTR: Naver 제공값 사용. 0이면 경쟁도별 기본값 (낮음=5%, 중간=3%, 높음=1.5%)
```

### KQS 등급

| 등급 | 점수 | 의미 |
|------|------|------|
| S | ≥ 15 | 검색량 높고 경쟁 낮음. 최우선 키워드 |
| A | ≥ 6 | 좋은 기회. 적극 활용 |
| B | ≥ 2 | 보통. 보조 키워드로 사용 |
| C | ≥ 0.5 | 낮은 효율. 여유 있을 때 |
| D | < 0.5 | 비효율적. 사용 비권장 |

### 출력
- KQS 점수순 정렬된 KeywordResult 배열
- 각 항목: keyword, volume, pc_volume, mobile_volume, comp_idx, avg_clk, avg_ctr, ad_depth, kqs_score, kqs_grade

## Step F: 이미지 필터 (엑셀 모드만)

### F-1. 이미지 → 한 줄 설명 (step2a_describeImage)

```
Gemini 프롬프트:

이 이미지에 보이는 상품을 한 줄로 설명하세요.
형식: "카테고리 + 핵심 특징 + 소재/색상"
예: "빨간색 충전식 전동 타카 공구"
예: "흰색 EVA 재질 논슬립 선반 매트"

한 줄 텍스트만 반환. 따옴표, 설명, 부연 없이.
```

- 이미지를 base64로 변환 후 Gemini Vision에 전송
- 최대 4MB, 10초 타임아웃
- temperature: 0.1 (정확도 우선)

### F-2. 키워드 vs 이미지 관련성 점수 (step2b_scoreKeywordsDetailed)

```
Gemini 프롬프트:

상품 이미지 설명: "{F-1 결과}"

아래 키워드 각각이 위 상품과 얼마나 관련 있는지 0~100 점수를 매기세요.
- 100: 상품 그 자체를 설명하는 키워드
- 85+: 상품의 용도, 기능, 소재, 설치 장소 등 직접 관련
- 50~84: 간접적 연관 (같은 카테고리이나 다른 제품)
- 0~49: 전혀 무관한 키워드

JSON 객체로만 반환. 예: {"키워드1": 95, "키워드2": 30}

키워드: [{Step E 통과 키워드들}]
```

### F-3. 필터링

- **85점 이상**: 통과 (IMAGE_RELEVANCE_CUTOFF = 85)
- **84점 이하**: 제거
- **Gemini가 빠뜨린 키워드**: 보수적으로 통과 (스코어 없는 건 살림)
- **파싱 실패** (scores 빈 객체): 보수적으로 전체 통과

## 캐시 시스템

### keyword_cache 테이블

| 컬럼 | 타입 | 설명 |
|------|------|------|
| keyword | text (PK) | 키워드 원문 |
| monthly_pc_qc_cnt | int | PC 월간 검색량 |
| monthly_mobile_qc_cnt | int | 모바일 월간 검색량 |
| monthly_qc_cnt | int | 합계 검색량 |
| comp_idx | text | 경쟁도 (높음/중간/낮음) |
| monthly_avg_clk_cnt | float | 평균 클릭수 |
| monthly_avg_ctr | float | 평균 CTR |
| pl_avg_depth | float | 광고 평균 깊이 |
| kqs_score | float | KQS 점수 |
| kqs_grade | text | KQS 등급 |
| last_refreshed_at | timestamptz | 마지막 갱신 |
| updated_at | timestamptz | 마지막 업데이트 |

### 동작

1. **읽기**: `naverFetchAllRelated()` 진입 시 `getCachedKeywords()`로 일괄 조회 (100개씩 배치)
2. **매칭**: 캐시에 있는 키워드 → 네이버 API 스킵, 캐시 데이터 사용
3. **API 호출**: 캐시에 없는 키워드만 네이버 API 호출
4. **쓰기**: API 응답을 `upsertKeywordCache()`로 저장 (100개씩 배치, onConflict: keyword)

### 필터 UI (v28~)

결과 테이블 위에 필터 바 표시:

- **등급 칩**: S / A / B / C / D (다중 선택, 토글)

```
A AI 75개 → B+C 네이버 확장 2340개 → 금칙어 1890개 → 검색량 420개 → 최종 380개
```
