## 1. 긴급 — Edge Function HTML 인라인 배포 에러 (5/26 마지막 작업)

### 배경
- Supabase Edge Function은 index.ts만 번들링함 (정적 HTML 파일 미포함)
- 그래서 `keyword-tool.html`을 `const HTML_UI = "..."` 문자열로 index.ts 안에 인라인 삽입 → GET 요청 시 서빙
- Python `json.dumps(html, ensure_ascii=False)`로 한글 보존한 문자열을 생성해서 index.ts에 넣음

### 현재 에러
```
unexpected deploy status 400: {"message":"Failed to bundle the function 
(reason: The module's source code could not be parsed: Expected unicode escape 
at file:///tmp/...index.ts:1149:37844)"}
```

- **원인**: HTML 문자열 안에 `\u`로 시작하는 문자 조합이 있어서 Deno 파서가 유니코드 이스케이프로 해석 시도 → 파싱 실패
- **수정 방법**: HTML 문자열을 `json.dumps`가 아니라 **템플릿 리터럴 (백틱)** 으로 감싸거나, `\u` 패턴을 이스케이프 처리해야 함
- 추천: `const HTML_UI = \`...\`;` (백틱 템플릿 리터럴) 사용 시 HTML 안의 백틱만 `\\\`` 이스케이프하면 됨. 또는 `String.raw` 활용

### 파일 위치
- `supabase/functions/keyword-tool/index.ts` — 1149행에 `const HTML_UI = "..."` 시작
- `supabase/functions/keyword-tool/keyword-tool.html` — 원본 HTML (800줄, 라이트 테마 + 전체 기능)
- 목업 원본: `docs/기획/dukwoo_site_v1.html` (937줄, 8페이지 인터랙티브)

### 해결 순서
1. `keyword-tool.html` 읽기
2. HTML 안의 백틱(`) → `\`` 이스케이프, `${` → `\${` 이스케이프
3. `const HTML_UI = \`...\`;` 형태로 index.ts 1149행 교체
4. `npx supabase functions deploy keyword-tool --no-verify-jwt --project-ref uifjabklkmvfbsplvxsu`
5. https://uifjabklkmvfbsplvxsu.supabase.co/functions/v1/keyword-tool 접속 확인

### 중요 디자인 규칙
- **목업 `dukwoo_site_v1.html`의 라이트 테마 그대로** — 흰 배경, 흰 카드, 골드(#D4A853) 악센트, 네이비(#0F172A) 네비바
- "자체 AI 엔진" 표기 (Gemini 노출 절대 금지)
