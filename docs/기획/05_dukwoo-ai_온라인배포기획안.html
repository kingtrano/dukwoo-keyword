Executive Summary

안형선 이사가 v35 키워드 도구에 만족, 베타 사용자 10명 대기 중. 핵심 전략: 당분간 수익보다 키워드 데이터 자산 축적에 집중하고, Gemini→Gemma 로컬 전환으로 변동비를 0에 수렴시킨다. 수집된 키워드 DB가 해자(moat)가 되어, 후발 주자가 따라올 수 없는 데이터 우위를 확보한다.

조사 결과
🔍 네이버 검색광고 API 약관
✅ 결론: 사용 가능 (단, 조건부)
  • 공식 약관: "광고주가 직접 또는 외주 업체를 통해 개발하는 경우 사용 가능" — 제3자 서비스 개발 명시 허용
  • 셀러노트, 아이템스카우트, 판다랭크 등 기존 서비스들이 동일 API로 수년간 운영 중 — 시장 선례 충분
  • 주의: API 결과 데이터를 "허용 범위를 넘어 무단 복제/배포"하는 것은 금지 → 우리는 키워드 생성 도구이고 검색량 데이터 자체를 재판매하는 게 아니므로 해당 없음
  • Rate limit: 공식 명시 없음 (호출량 제한 없다는 커뮤니티 증언), 마스터 계정 1,000개 광고계정 연결 가능
  • 우리 전략과 시너지: 키워드 캐시 DB가 쌓이면 네이버 API 호출이 자연 감소 → 월 1회 검색량 일괄 업데이트만 하면 됨
🤖 Gemma 로컬 대체 가능성
⚠️ 결론: 가능하나 품질 테스트 필요
  • gemma4:e4b (4B, 9.6GB) — 현재 맥북에 설치 완료. 140개국어 지원, 한국어 포함. JSON structured output 네이티브 지원
  • gemma4:26b (26B MoE, 17GB) — 이미 설치됨. MMLU Pro 82.6%로 e4b(69.4%) 대비 큰 성능 향상
  • Mac Studio M4 Max 128GB (8월 도착) → gemma4:26b를 여유 있게 돌릴 수 있음. 31B Dense도 가능
  • 리스크: 한국어 SEO 키워드 생성 품질은 Gemini 2.5-flash 대비 열화 가능성 있음 → A/B 테스트 필수
  • 전환 전략: 즉시 전환이 아니라 Gemini로 운영하면서 Gemma 품질 테스트 → 80% 이상 적중률 확인 후 전환
🌐 keywordlab.ai 도메인
🔎 결론: 등록 가능 여부 직접 확인 필요
  • .ai 도메인은 누구나 등록 가능 (국적/자격 제한 없음)
  • 등록 비용: 연 $20~80 (Cloudflare 기준 최저가)
  • dukwootrading.com → keywordlab.ai 전환은 브랜드 리포지셔닝으로 적절 (무역회사 → IT 기업 이미지)
  • 즉시 확인: Cloudflare 또는 Namecheap에서 keywordlab.ai 검색 → 가능하면 바로 등록 권장

🔄 월 1회 검색량 업데이트 전략

회장님 아이디어대로, 이미 수집된 키워드는 월 1회만 네이버 API로 검색량을 갱신합니다.

  • 실시간 조회 대상: 신규 상품의 키워드만 (캐시에 없는 것)
  • 월간 배치 업데이트: 기존 캐시 키워드의 검색량을 월 1회 일괄 갱신
  • 효과: 6개월 후 일일 API 호출량 70% 감소 → rate limit 걱정 없음
  • 추가 자산: 시계열 검색량 데이터 축적 → "이 키워드 트렌드가 올라가고 있다" 분석 가능 (향후 프리미엄 기능)

왜 이게 해자인가: 후발 주자가 동일한 도구를 만들어도, 우리는 이미 300만 키워드 × 검색량 × 카테고리 매칭 데이터를 보유. 새로 시작하면 이 데이터를 모으는 데만 1년 걸림. 또한 "어떤 카테고리에 어떤 키워드가 실제로 매출에 기여했는지" 사용자 피드백까지 쌓이면 단순 API 호출로는 재현 불가.
