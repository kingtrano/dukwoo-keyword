각 단계에서 뭐가 수동이고, 뭘 자동화할 수 있는지

1

상품 만들기

도매처에서 가져와서 네이버에 올릴 수 있는 상태로 가공

작업
지금 (Before)
자동화 (After)
판정
도매처 크롤링51곳, 197만 SKU
유플랫 자동월 79.8만원, 자동 크롤링+재고연동+가격변동 반영
그대로 유지51곳 크롤링을 직접 만드는 건 비현실적. 유플랫이 하는 게 맞음.
유지
카테고리 매핑도매처 → 네이버 카테고리
자체 프로그램 (자동 90%)네이버 API 사용, 나머지 10% 수동
그대로 유지이미 90% 자동. 10%를 LLM으로 올릴 수 있으나 효과 미미.
유지
상품명 수정검색 최적화 상품명 생성
GPT/Gemini API 건별 호출자체 프로그램으로 하나씩 호출. 비용 건당 ~2-3원.
배치 API로 전환동일 품질, API 비용 50% 절감. 코드 수정만으로 적용.
자동화
키워드 선정1~40개 키워드 세팅
자체 프로그램 + 수동 입력네이버광고 API로 키워드 조합은 반자동. 유플랫에 세팅은 수동.
조합 자동화, 입력은 수동추천 키워드 자동 생성+순위화 가능. 유플랫에 넣는 건 사람이 해야 함.
반자동
이미지 가공리사이즈 + 워터마크
알바가 캡처툴로 수동건당 50원. 상세페이지는 도매처 원본 그대로.
Python Pillow 완전 자동리사이즈+워터마크 일괄 처리. 비용 거의 0원. 바로 적용 가능.
자동화
속성값 입력색상, 사이즈, 재질 등
알바가 유플랫에서 수동 입력건당 80~100원. 가장 큰 비용.
LLM이 추출, 입력은 수동상품 설명에서 속성값 자동 추출 → 엑셀 생성까지 자동. 유플랫에 넣는 건 수동. 복붙이라 시간은 줄어듦.
반자동
2

등록하기

가공 완료된 상품을 네이버 스마트스토어에 올리기

작업
지금 (Before)
자동화 (After)
판정
네이버 등록건당 4~5초, 일 2만건
유플랫 반자동카테고리 매칭 후 바로 등록. 이미 빠름.
그대로 유지4~5초/건이면 충분히 빠름. 건드릴 이유 없음.
유지
재고 연동도매처 재고 실시간 반영
유플랫 자동 크롤링도매처 재고 변동 자동 반영.
그대로 유지유플랫이 잘 하고 있음. 직접 구현하면 51곳 각각 대응 필요.
유지
가격 변동 반영매일 발생
유플랫 자동 크롤링매일 도매처 가격 변동 자동 반영.
그대로 유지동일. 유플랫 역할.
유지
3

관리하기

등록된 상품의 성과 분석, 순환(올리고 내리기), 주문/정산

작업
지금 (Before)
자동화 (After)
판정
판매 데이터 분석CTR, 전환율, 키워드
매출만 확인 (수동)샵마인→구글시트. 클릭률/전환율/키워드 미분석.
자동 대시보드네이버 API로 일별 자동 수집 → 구글시트 대시보드. 안형선 이사도 계획 중이던 것.
자동화
상품 순환 판단내릴 상품 / 올릴 상품
등급 변화 시 수동 판단기준이 정성적. 데이터 근거 부족.
스코어링으로 추천 리스트대시보드 데이터 기반 후보 자동 생성. 최종 결정과 실행(등록/삭제)은 사람이.
반자동
키워드 성과 추적어떤 키워드가 잘 되나
미분석키워드별 성과 데이터 수집 안 함.
자동 수집 + 피드백어떤 키워드가 클릭/전환 높은지 자동 추적. 키워드 교체 추천.
자동화
주문 확인 / 발주일 100건
유플랫(UOS) 자동주문 확인, 발주, 배송, 송장 등록 모두 UOS 처리.
그대로 유지이미 자동화되어 있음.
유지
정산 확인일반 + 빠른정산
샵마인 통계 확인일반정산(8일 후) + 빠른정산(익일). 수동 확인.
그대로 유지대시보드에 정산 데이터 포함시킬 수 있으나, 핵심 병목 아님.
유지

CS 하기

고객 문의 응대, 반품/교환 처리

작업
지금 (Before)
자동화 (After)
판정
고객 문의 응대상품/재고/발송 문의
2명이 수동, 템플릿 있음김수인+김자헌. 10건/일. 샵마인/문자/전화.
현행 유지 권장하루 10건에 AI 시스템 구축은 과잉. 기존 템플릿으로 충분. 볼륨이 50건+ 넘으면 재검토.
유지
반품/교환도매처별 절차 상이
건별 수동 처리도매처마다 접수방법 다름. 1~2주 소요.
현행 유지도매처 절차 DB는 만들 수 있지만, 반품 자체가 전화/문자로 처리되므로 자동화 효과 제한적.
유지

전체 요약

자동화 가능 (4건)
반만 가능 (3건)
현행 유지 (6건)
✅ 바로 하면 되는 것 이미지 자동 가공, 상품명 배치 전환, 판매 대시보드, 키워드 성과 추적
⚡ 반만 되는 것 키워드 선정, 속성값 추출, 순환 판단 — 생성은 자동, 유플랫 입력은 수동
— 건드릴 필요 없는 것 크롤링, 카테고리(90%), 등록, 재고, 가격, 주문/발주, 정산, CS, 반품
🔑 이 한 가지를 확인하면 "반만"이 "완전"으로 바뀔 수 있음

안형선 이사에게: "유플랫에 엑셀로 상품 데이터를 대량 업로드(임포트)할 수 있습니까?"
→ 된다면: 속성값/키워드를 자동 생성 → 엑셀로 만들어서 → 유플랫에 한번에 올리기 가능
→ 안 된다면: 생성은 자동이지만 입력은 사람이 하나씩 해야 함
