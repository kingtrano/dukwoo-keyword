## 1. Colab 상품명 최적화 (완성 4/18)

### 개요
- **목적:** 덕우무역 네이버 스마트스토어 상품명 SEO 최적화
- **대상 사용자:** 안형선 이사 (비개발자 — Google Colab 노트북으로 제공)

### 핵심 로직
1. UFlat 엑셀 업로드 (B열=묶음코드, D열=상품명, L열=키워드, M열=이미지URL)
2. 상품명에서 핵심 단어 추출 → 쉼표 구분 → 네이버 검색광고 API 호출
3. 연관 키워드 중 관련성 필터링 (2글자 청크 매칭 2개 이상)
4. 원본 상품명에 이미 있는 단어와 겹치는 키워드는 제외 (중복 방지)
5. 상품당 최대 40개 키워드 풀 생성 → 랜덤 2개 골라 상품명 뒤에 붙임
6. 결과: D열 교체 + L열 키워드 채움 → 등록용 엑셀 다운로드

### 두 가지 방법
- **Method A (이미지 기반):** Gemini Vision → 상품 대명사 추출 → 키워드 검색
  - 노트북: `naver_keyword_pipeline.ipynb`
- **Method B (엑셀 기반):** 상품명에서 직접 키워드 추출 ← 주력
  - 노트북: `naver_title_generator_v5.ipynb` (최신)

## 2. Keyword-Tool Edge Function (v35, 2026-04-22)

### 개요
- **Supabase 프로젝트:** uifjabklkmvfbsplvxsu
- **함수명:** keyword-tool
- **소스:** `~/.openclaw/edge-functions/supabase/functions/keyword-tool/index.ts`
- **프론트엔드:** `~/.openclaw/workspace/docs/keyword-tool.html`
- **용도:** 상품명 → AI 키워드 생성 + 기계적 조합 → 3단 필터 → 검색량 체크 → KQS 품질 등급 산출

### 파이프라인 (v35 — excel 모드)
```
Step A1: Gemini 2.5-flash (temperature 1.0, thinkingBudget:0)
  → 상품명 파싱: extractCoreKeywords() — 브랜드·모델번호·용량 제거
  → 6패턴 프롬프트 (용도/재료/동력/형태/대상/동의어) + 8규칙
  → 30~50개 키워드 생성

Step A2: 핵심어+동의어+수식어 추출 → 기계적 조합
  → A2-1: Gemini → {coreWords, synonyms} JSON
  → A2-2: Gemini → 수식어 40~60개 (형태접미어 "건,총,기" 포함)
  → A2-3: generateCompounds() — allCores(핵심어+동의어) × 수식어, 2/3어절 양방향 → 최대 500개
  → A1+A2 병합 → ~530개

Step B: 3단 필터 (v35 신규)
  → ① 금칙어 필터 (BANNED_WORDS + seller brand + BANNED_BRANDS)
  → ② isOriginalSubstring() — 상품명의 연속 부분문자열 제거 (6글자↑만 체크)
  → ③ 짧은 키워드 필터 — 공백 제거 후 2글자 이하 제거
  → 로그: [filter] banned=N, origSub=N, short=N

Step C: 네이버 검색량 체크 (볼륨만, 확장 없음)
  → 검색광고 API (HMAC-SHA256 서명)
  → keyword_cache 테이블 캐싱 (읽기+쓰기)
  → PC >= 10 OR 모바일 >= 10, 총합 <= 50,000
  → KQS 등급 산출 (S/A/B/C/D)

Step D: 이미지 필터 (엑셀 모드만, 2단계)
  → step2a: Gemini Vision → 이미지 한 줄 설명
  → step2b: 키워드 vs 이미지설명 관련성 스코어링 (0~100, cutoff=85)
```

### Single 모드
A1 → A2 → 병합 → 3단 필터 → 네이버 볼륨 (이미지 필터 없음)

### A1 프롬프트 규칙 (v35)
1. 패턴당 5~10개, 총 30~50개
2. 띄어쓰기 없이 붙여쓰기
3. 브랜드명/모델번호 제외
4. 단독 수식어 금지 ("충전" 단독 ✕, "충전타카" ✓)
5. 실제 네이버에서 검색될 자연스러운 조합만
6. **동일 카테고리 키워드만** (타카→액정필름 ✕)
7. **3글자 이상 조합만** (2글자 이하 ✕)

### isOriginalSubstring() — v35 신규
- 키워드(공백 제거·소문자) 6글자 이상이 상품명(공백 제거)에 연속 포함되면 제거
- 역순 조합은 통과 (상품명에 연속으로 포함 안 됨)
- 예: "충전실타카"는 상품명 부분문자열이면 제거, "실타카충전"은 통과

### 핵심 함수들
- `extractGeminiText(data)` — Gemini 2.5 thinking 파트 건너뛰고 실제 텍스트만 추출
- `step1_expandKeywords()` — A1: Gemini 키워드 생성 (6패턴+8규칙)
- `step1b_extractModifiers()` — A2: 핵심어/동의어/수식어 추출 + 조합
- `generateCompounds()` — A2-3: 전수 조합 (2/3어절, 양방향, 500캡)
- `isOriginalSubstring()` — 원본 서브스트링 필터
- `containsBanned()` — 금칙어/브랜드 3단 필터
- `step3_checkVolume()` — 네이버 볼륨 체크 (캐시 읽기+쓰기)
- `step2_filterByImage()` — 이미지 필터 (step2a+step2b)

## 공통 기술 참조
- **네이버 검색광고 API:** `https://api.searchad.naver.com/keywordstool`
  - HMAC-SHA256 서명 (base64 인코딩, hex 아님)
  - hintKeywords 쉼표 구분 필수
  - customer_id: 3062209
- **검색량 필터:** 최소 10, 최대 50,000
- **Gemini 모델:** gemini-2.5-flash → gemini-2.5-flash-lite (fallback), 각 3회 retry
- **Gemini thinking 비활성화:** 모든 callGemini 호출에 `thinkingConfig: { thinkingBudget: 0 }` 필수
- **Supabase MCP 배포:** `deploy_edge_function` (verify_jwt: false)
