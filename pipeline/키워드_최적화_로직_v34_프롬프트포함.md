# 덕우 키워드 최적화 도구 — 로직 정리

> 최종 업데이트: 2026-04-22 (v34 / 배포 완료)
> 이전 버전: v13 (4/21) → v28~v29 (A2 조합엔진) → v33 (관련성필터) → **v34** (Gemini-only, 네이버확장 제거)

---

## 전체 흐름

```
상품명 + 이미지URL 입력
  → Step A1: Gemini AI 키워드 생성 (30~50개, 6패턴)
  → Step A2: Gemini 핵심어+동의어+수식어 추출 + 기계적 조합 (최대 500개)
  → A1 + A2 병합 (약 520개)
  → Step B: 금칙어/브랜드 필터
  → Step C: 네이버 검색량 체크 (확장 없음, 볼륨만 조회)
  → Step D: 이미지 필터 (엑셀 모드만, Gemini 비전 85점+)
  → 최종 키워드 리스트 + KQS 등급
```

### v33 → v34 핵심 변경

| 항목 | v33 | v34 |
|------|-----|-----|
| 키워드 생성 | Gemini + **네이버 2라운드 확장** | **Gemini-only** (네이버 확장 완전 제거) |
| A1 프롬프트 | 11개 카테고리 (55~80개) | **6개 패턴** (30~50개, 용도/재료/동력/형태/대상/동의어) |
| A2-1 출력 | 단순 배열 `["실타카"]` | **구조화 JSON** `{coreWords, synonyms}` |
| A2-2 수식어 | 기본 카테고리 | **형태접미어** 추가 (건, 총, 기, 기계, 공구, 세트) |
| 조합 코어 | coreWords만 | **coreWords + synonyms 전체** |
| 조합 캡 | 250개 | **500개** |
| 관련성 필터 | extractRelevanceTerms 있음 | **삭제** (네이버 확장 없으므로 불필요) |
| 네이버 역할 | 확장 + 검색량 | **검색량 체크만** |
| 안형선 적중률 | 3/10 (30%) | **10/10 (100%)** |

### 왜 네이버 확장을 제거했나?

네이버 `/keywordstool` API가 입력 키워드와 **같은 광고 카테고리** 전체를 반환함.
- "실타카" 검색 → 제빙기, 믹서기, 커피머신 등 동일 광고 카테고리 수천 개 반환
- v33에서 관련성 필터를 추가했지만 근본 해결이 안 됨 (히트율 30%)
- **해결:** 네이버는 검색량 체크만 하고, 키워드 생성은 Gemini가 전담

## Step A1: Gemini AI 키워드 생성

### A1-1. 상품명 파싱 (extractCoreKeywords)

입력 예시: `20V MAX 23GA 충전 실타카 본체 베어툴 DCN623N-KR`

5단계 순차 정규식 처리:

1. **대괄호 제거** — `[덕우무역]` 등 브랜드/판매자 표기 제거
2. **모델번호 제거** — 영문+숫자 조합 (DCN623N-KR, AB-1234 등)
3. **영문 단어 필터** — 단독 영단어 제거. 허용 목록: pro, max, mini, plus, lite, ultra, set, kit, led, usb, dc, ac
4. **용량/수량 제거** — 500mm, 800ml, 100매, 10개, 3P 등
5. **토큰화** — 2자 이상 한글 단어만 추출, 중복 제거

파싱 결과: `["충전", "실타카", "본체", "베어툴"]`

### A1-2. Gemini 프롬프트 (v34)

```
당신은 네이버 스마트스토어 SEO 전문가입니다.

상품명: "{상품명 원문}"
핵심 키워드 힌트: [{파싱된 토큰들}]

이 상품을 네이버에서 검색할 때 소비자가 실제로 입력할 수 있는 키워드를 30~50개 생성하세요.

반드시 포함할 패턴:
1. [용도]+[상품명]: 목공타카, 인테리어선반
2. [재료]+[상품명]: 나무타카, 원목선반
3. [동력/방식]+[상품명]: 충전타카, 무선드릴
4. [상품명]+[형태]: 실타카건, 실타카총, 타카기
5. [대상/직업]+[상품명]: 목수타카, 초보드릴
6. 동의어/유사어: 핀네일러, 스테이플건

규칙:
1. 띄어쓰기 없이 붙여쓰기
2. 브랜드명/모델번호 제외
3. 단독 수식어 금지 — "충전" 단독은 안됨, "충전타카"는 OK
4. 원본 상품명에 이미 있는 단어 조합은 제외

JSON 배열로만 반환. 예: ["목공타카", "충전실타카"]
```

### A1-3. Gemini 호출 설정

- 모델: gemini-2.5-flash (실패 시 gemini-2.5-flash-lite fallback)
- 각 모델 최대 3회 retry (429→3/6/9초 대기, 500+→2/4/6초 대기)
- temperature: 1.0 (다양성 극대화)
- maxOutputTokens: 4096
- **thinkingConfig: { thinkingBudget: 0 }** (thinking 비활성화 — v31부터 필수)

### A1-4. 응답 파싱

1. **thinking 파트 제거** — Gemini 2.5는 `{thought: true}` 파트가 섞여 나옴. 뒤에서부터 스캔해서 thought 아닌 마지막 텍스트만 추출
2. **JSON 배열 추출** — 코드블록 마커 제거 후 `[...]` 패턴 매칭 → JSON.parse

### A1-5. 후처리

1. Gemini 결과에서 문자열 2자 이상만 필터
2. 파싱된 핵심 토큰을 뒤에 추가
3. Set으로 중복 제거
4. 최대 100개 cap

## Step A2: 핵심어+동의어+수식어 추출 + 기계적 조합

> **목적:** A1은 완성된 키워드를 생성하지만 빠지는 조합이 있음. A2는 **수식어만 뽑고 코드가 전수 조합**하여 누락 방지.

### A2-1. 핵심어 + 동의어 추출 (v34 변경)

**Gemini 프롬프트:**

```
상품명: "{상품명 원문}"
핵심 키워드 힌트: [{파싱된 토큰들}]

이 상품의 핵심 상품 카테고리 명사를 1~2개만 뽑으세요.
그리고 그 핵심어의 동의어/유사어도 함께 뽑으세요.

JSON으로 반환:
{"coreWords": ["실타카"], "synonyms": ["타카", "타카건", "네일건", "핀네일러", "스테이플건"]}
```

- temperature: 0.2 (정확도 우선)
- maxOutputTokens: 1024

**응답 파싱:** JSON.parse로 `{coreWords, synonyms}` 객체 추출

### A2-2. 수식어 추출 (v34 변경)

**Gemini 프롬프트:**

```
상품명: "{상품명 원문}"
핵심 상품어: [{A2-1 핵심어들}]

이 상품에 붙일 수 있는 수식어를 단어 하나씩만 나열하세요.

카테고리:
1. 용도 (목공, 인테리어, DIY, 가정용, 업소용...)
2. 방식/동력 (충전, 무선, 전동, 에어, 수동...)
3. 소재/규격 (스테인리스, 대형, 소형, 미니...)
4. 대상/상황 (초보, 전문가, 선물...)
5. 기능/특징 (경량, 접이식, 방수, 휴대용...)
6. 형태접미어 (건, 총, 기, 기계, 공구, 세트, 장비, 용품)

규칙:
- 한 단어씩만 (띄어쓰기 없이)
- "건", "총" 같은 1글자 수식어도 반드시 포함
- 브랜드명/모델번호 제외
- 총 40~60개
- JSON 배열로만 반환
```

- temperature: 0.8
- maxOutputTokens: 2048

### A2-3. 기계적 조합 (generateCompounds — v34 변경)

**v34 핵심:** coreWords + synonyms **전체**를 코어로 사용하여 조합

```
입력:
  coreWords: ["실타카"]
  synonyms: ["타카", "타카건", "네일건", "핀네일러", "스테이플건"]
  modifiers: ["충전", "무선", "목공", "DIY", "건", "총", "기", ...]

  allCores = coreWords ∪ synonyms = ["실타카", "타카", "타카건", "네일건", ...]

조합 규칙:
  1. 2단어 앞: modifier + core (모든 수식어 × 모든 코어)
     → "충전실타카", "충전타카", "충전네일건", "목공타카", "목공네일건", ...
  
  2. 2단어 뒤: core + modifier (모든 코어 × 모든 수식어)
     → "실타카건", "실타카총", "타카기", "타카기계", ...
  
  3. 3단어: modifier_A + modifier_B + core (상위 12개 수식어 × 상위 12개 × 코어)
     → "충전무선실타카", "목공전동타카", ...

  중복 제거 후 최대 500개 cap
```

### A2-4. A1 + A2 병합

```
allKeywords = A1 결과 ∪ A2 compounds ∪ A2 synonyms
Set으로 중복 제거
→ 약 520개 (실타카 테스트 기준)
```

## Step C: 네이버 검색량 체크 (v34 — 확장 없음)

> **v34 변경:** 네이버 API는 **검색량 조회만** 수행. 키워드 확장(연관 키워드 수집)은 **완전 제거**.

### 동작

금칙어 필터 통과한 약 520개 키워드를 네이버 광고 API `/keywordstool`에 5개씩 배치로 넣어 검색량만 확인.

### 캐시 시스템

- `keyword_cache` Supabase 테이블에서 먼저 조회 (100개씩 배치)
- 캐시 히트 → 네이버 API 스킵
- 캐시 미스 → 네이버 API 호출 → 결과를 캐시에 저장 (upsert)

### API 호출

```
GET https://api.searchad.naver.com/keywordstool
  ?hintKeywords={5개 콤마 구분}&showDetail=1

Headers:
  X-Timestamp: {밀리초}
  X-API-KEY: {네이버 광고 API 키}
  X-Customer: {고객 ID}
  X-Signature: HMAC-SHA256(timestamp.GET./keywordstool)
```

### 필터 조건

```
통과 조건:
  (PC 월간 검색량 ≥ 10  OR  모바일 월간 검색량 ≥ 10)
  AND
  (PC + 모바일 합계 ≤ 50,000)
```

- "< 10" 문자열 → 0으로 처리
- 검색량 0인 키워드 → 제거
- 50,000 초과 → 제거

### KQS (Keyword Quality Score) 산출

```
KQS = (검색량 × CTR × 2) / (경쟁도 × 광고깊이)

경쟁도 매핑: 높음=3, 중간=2, 낮음=1
광고깊이: 최소 0.5 (0일 경우)
CTR: Naver 제공값 사용. 0이면 경쟁도별 기본값 (낮음=5%, 중간=3%, 높음=1.5%)
```

### KQS 등급

| 등급 | 점수 | 의미 |
|------|------|------|
| S | ≥ 15 | 검색량 높고 경쟁 낮음. 최우선 키워드 |
| A | ≥ 6 | 좋은 기회. 적극 활용 |
| B | ≥ 2 | 보통. 보조 키워드로 사용 |
| C | ≥ 0.5 | 낮은 효율. 여유 있을 때 |
| D | < 0.5 | 비효율적. 사용 비권장 |

## Step D: 이미지 필터 (엑셀 모드만)

### D-1. 이미지 → 한 줄 설명 (step2a_describeImage)

```
Gemini 프롬프트:
이 이미지에 보이는 상품을 한 줄로 설명하세요.
형식: "카테고리 + 핵심 특징 + 소재/색상"
예: "빨간색 충전식 전동 타카 공구"
한 줄 텍스트만 반환.
```

- base64 변환 후 Gemini Vision 전송
- 최대 4MB, 10초 타임아웃
- temperature: 0.1

### D-2. 키워드 vs 이미지 관련성 점수 (step2b_scoreKeywordsDetailed)

```
Gemini 프롬프트:
상품 이미지 설명: "{D-1 결과}"
아래 키워드 각각이 위 상품과 얼마나 관련 있는지 0~100 점수를 매기세요.
JSON 객체로만 반환. 예: {"키워드1": 95, "키워드2": 30}
키워드: [{Step C 통과 키워드들}]
```

### D-3. 필터링

- **85점 이상**: 통과 (IMAGE_RELEVANCE_CUTOFF = 85)
- **84점 이하**: 제거
- **Gemini가 빠뜨린 키워드**: 보수적으로 통과
- **파싱 실패**: 보수적으로 전체 통과

```
step1_count    = A1 Gemini 키워드 수
step1b_count   = A2 조합 키워드 수
merged_count   = A1 + A2 + synonyms 병합 (중복 제거)
filter_count   = 금칙어 필터 통과
naver_count    = 검색량 필터 통과
image_count    = 이미지 필터 통과 (엑셀 모드만)
```
